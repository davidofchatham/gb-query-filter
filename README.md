# GB Query Filter

GB Query Filter is a lightweight WordPress plugin that adds customizable
front-end filters to the GenerateBlocks Query Loop block. It enables
live filtering based on taxonomies, custom fields, sort order, and
search terms while maintaining full compatibility with GenerateBlocks
2.0+.

The plugin integrates with a custom block that links directly to the
Query Loop, supports optional AJAX filtering, and syncs all filter
states to the URL for shareable filter results.

## Features

### Front-End Filters

Supports filtering by:
- Taxonomies (built-in and custom)
- Meta Box Custom Fields
- Advanced Custom Fields (ACF)
- Search Term

Filter control types include:
- Select dropdowns
- Checkboxes (with multi-select support)
- Radio buttons
- Text inputs

Additional filter features:
- Multiple selections
- Auto-apply mode
- Reset button
- AJAX filtering

### Block-Based Integration

The plugin registers a custom block that connects to GenerateBlocks'
Query Loop. Users can configure:
- Post types
- Taxonomy filters (categories, tags, custom taxonomies)
- Meta Box field filters
- ACF field filters
- Search functionality
- Control types (select, radio, checkboxes, text)
- Front-end filter behavior
- AJAX vs. full-page reload mode
- Auto-apply vs. manual apply button

### URL Parameter Sync

Filter states are synced to the URL using the GB Query Filter naming
pattern (shown here in the flat format used when no target ID is set; see
[URL Structure](#url-structure) for the scoped format):

    ?gbqf_search=steel+railings
    ?gbqf_cat[]=1&gbqf_cat[]=2
    ?gbqf_tag[]=5&gbqf_tag[]=7
    ?gbqf_tax[category][]=10&gbqf_tax[category][]=15
    ?gbqf_meta[color]=blue
    ?gbqf_meta[project_status]=active
    ?gbqf_meta[colors][]=red&gbqf_meta[colors][]=blue

On page load, the plugin restores all filter states from the URL, making
filters shareable and bookmarkable.

### Optional AJAX Support

Two filtering modes are available:
1. Full page reloads
2. AJAX filtering and pagination (if enabled)

## Installation

### From GitHub

1.  Download the repository ZIP.
2.  Upload it to WordPress under Plugins → Add New → Upload Plugin.
3.  Activate the plugin.
4.  Add the GB Query Filter block to any page containing a
    GenerateBlocks Query Loop.

## Usage

1.  Create or edit a page with a GenerateBlocks Query Loop block.
2.  Add the GB Query Filter block above the Query Loop.
3.  Configure the filter settings:
    -   Link filter to Query Loop (set target ID)
    -   Enable search field
    -   Choose taxonomies (categories, tags, custom)
    -   Add Meta Box fields (if Meta Box plugin is active)
    -   Add ACF fields (if ACF plugin is active)
    -   Select control types (auto, select, radio, checkboxes, text)
    -   Enable or disable AJAX filtering
    -   Enable or disable auto-apply mode
4.  Publish or update the page.

## URL Structure

Examples of URL parameters generated by the plugin:

**Search term**

    ?gbqf_search=modern+stairs

**Category filters**

    ?gbqf_cat[]=1&gbqf_cat[]=2

**Tag filters**

    ?gbqf_tag[]=5&gbqf_tag[]=7

**Custom taxonomy filters**

    ?gbqf_tax[project_type][]=10&gbqf_tax[project_type][]=15

**Meta Box field filters**

    ?gbqf_meta[price]=100-200
    ?gbqf_meta[color]=blue

**ACF field filters**

    ?gbqf_meta[project_status]=active
    ?gbqf_meta[colors][]=red&gbqf_meta[colors][]=blue

The examples above use the flat parameter format, applied when the filter
block has no target ID set. When a target ID is set—required to isolate
multiple filters on the same page—the parameters are nested under that
target:

    ?gbqf[my-projects][search]=modern+stairs
    ?gbqf[my-projects][cat][]=1&gbqf[my-projects][cat][]=2
    ?gbqf[my-projects][meta][color]=blue

Resetting a filter clears only its own parameters.

## Developer Notes

-   Uses native WordPress block APIs and WP_Query modifications.
-   Does not modify GenerateBlocks core.
-   Filters only affect the inner post query of the Query Loop block.
-   Compatible with caching plugins when AJAX is disabled.
-   Pure PHP + vanilla JavaScript (no build process required).
-   Optional integrations: Meta Box and Advanced Custom Fields (ACF).
-   ACF checkbox fields use serialized array matching for multi-select support.
-   Designed for extensibility and future integrations.
-   Not affiliated with GeneratePress, GenerateBlocks, GenerateOne, Meta Box, or Advanced Custom Fields
-   Use at your own risk
